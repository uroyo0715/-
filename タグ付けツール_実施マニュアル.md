# 印象評価（SD法）タグ付けツール 実施マニュアル

スギ屋外暴露材（SC-2 未処理 / SC-4 熱処理）の経年変化画像に対し，人の印象（味・汚れ・経年など）を収集するためのツールの操作・運用手順をまとめる。実験を担当する学生がこのまま読んで実施できることを目的とする。

---

## 0. 全体像（最初に読む）

- ツールは **`タグ付けツール.html`** をブラウザで開くだけで動く（インターネット接続不要・オフライン可）。
- 役割は2つ。**実験者**（実施を仕切る学生）が最初に「実験設計」を1回決めて固定し，**評価者**（回答する人）が1人ずつ評価する。
- データは **1人が1セッション終えるごとに，その人専用のCSVが1ファイル**ダウンロードされる。全員ぶんが自動で1ファイルに集まるわけではない。最後に個別CSVを集めてまとめる。
- 評価者には，画像の試料名・処理・経過週は一切見せない（先入観を避けるため）。

---

## 1. 準備（実験者）

### 1.1 必要なファイル一式

ツールは画像を相対パスで参照する。**必ず以下を同じ場所にそろえて**配布・設置する。

```
（配布フォルダ）/
├─ タグ付けツール.html
├─ SC-2_cropped/   … 未処理材のクロップ済み画像（SC-2-0.jpg 〜 SC-2-104.jpg）
└─ SC-4_cropped/   … 熱処理材のクロップ済み画像（SC-4-0.jpg 〜 SC-4-104.jpg）
```

`タグ付けツール.html` 単体だけを送ると画像が表示されない。**フォルダごと（またはzipにまとめて）配布する。**

### 1.2 動作確認

1. `タグ付けツール.html` をダブルクリックしてブラウザで開く。
2. 「STEP1」の設計画面が表示されればOK。後述の手順で1度通しで試し，画像が正しく表示されることを確認する。

### 1.3 配布の2方式

| 方式 | 内容 | 向く場面 |
|------|------|----------|
| 共有1台方式 | 1台のPCで評価者が順番に回答 | 同じ場所に集まって実施できるとき |
| 各自PC方式 | 上記フォルダ一式を各評価者に配布し，各自のPCで回答→CSVを回収 | 離れた場所・各自の都合で実施するとき |

## 2. 実験設計を決める（実験者・最初に1回だけ）

ツール上部「STEP 1 実験設計」で設定し，**「この設計で確定して評価へ進む」**を押すと設計が固定される（以降，全評価者で同じ設定のまま実施する。途中で変えない）。

### 2.1 各設定の意味

| 設定 | 意味 |
|------|------|
| 割り当てモード | **セット指定**＝全76枚を固定セットに分割し各評価者にセット番号を割り当て（網羅保証・推奨）／**分担設計**＝ランダム層化で一部抽出／**代表ピックアップ**＝13週×2＝26枚／**全数**＝76枚すべて |
| 1人/1セットの枚数 | 1人が評価する枚数。セット指定では「1セットの枚数」になり，分割数 K＝ceil(76÷枚数) が決まる（既定26→3セット） |
| 各画像の目標評価人数 R | 1枚あたり何人に評価してほしいか（既定3）。信頼性のため複数人で評価する |
| 一貫性チェック用の重複枚数 | **同じ画像を後半でもう一度出す枚数**（既定3）。同じ人の評価のブレ（再現性）を測るため。どれが再掲かは評価者に分からない |
| 1枚あたり所要（秒） | 所要時間の目安計算用 |

設定を変えると，画面に**必要な延べ評価数・必要評価者数・1セッション所要時間**が自動計算される。これを見て人数と枚数を決める。

### 2.2 設計の考え方（疲労対策）

- 76枚を1人に全部やらせると疲労・基準のブレが出やすい。**分担設計で1人20〜30枚程度**にし，複数人で全76枚をカバーするのが基本。
- 例：1人24枚・各画像3人なら，延べ228評価・**必要評価者 約10人**・1人あたり約10〜12分。
- 各画像が目標R人に達したかは，後述「進捗確認」で随時チェックできる。

### 2.3 セット指定モードの運用（網羅を保証したいとき・推奨）

全76枚を固定の K セットに**決定的に分割**する（週・試料が偏らないよう自動配分）。各評価者にセット番号を割り当て，各セットを R 人ずつ評価すれば，**全画像がちょうど R 回覆われる**（網羅が保証される）。

- 必要評価者数 = **K × R**（例：1セット26枚 → K=3，R=3 なら 9人）。
- 実験者は「誰がどのセットか」の**名簿だけ**管理すればよい（例：A・B・Cさん→セット1，D・E・Fさん→セット2，G・H・Iさん→セット3）。
- 分割は決定的なので，各自のPCで開いても**同じセットが再現**される（分散配布でも崩れない）。
- 評価者はSTEP2で自分の**セット番号**を選んでから開始する。どのセットかはCSVの `set_id` 列に記録される。
- ランダムの分担設計と違い「終盤に特定画像だけ埋まらない」問題が起きず，無駄打ちもない。

## 3. 評価のやり方（評価者ごとに繰り返す）

設計を確定すると「STEP 2 評価の実施」が現れる。評価者1人ごとに次を行う。

1. （セット指定モードのとき）まず**自分のセット番号を選ぶ**（実験者の割り当てに従う）。続けて**評価者IDを入力**（氏名・イニシャル・S01 など）。**人ごとに必ず別のID**にする。
2. 「評価を開始」を押す。
3. **教示画面**が出る。読み上げ／黙読で内容を確認し，「評価を始める」を押す。
4. **練習2枚**（記録されない）でスケールの幅をつかむ。
5. 本番。左の**参照基準（設置直後・0週）**と中央の**評価対象**を見比べ，右の各項目を**7段階**で選ぶ。
   - キーボードの **1〜7** でも入力可。全項目に答えると「次へ」が押せる。
   - **約10枚ごとに休憩**が入る（進捗バーの赤い縦線が休憩位置）。
6. 最後まで答えると完了画面になる。**「結果をCSVで保存」**を押してCSVをダウンロードする。
7. 共有1台方式で次の人に渡すときは，完了画面の**「次の評価者へ」**を押す（前の人のデータはクリアされる）。

### 3.1 評価項目（5項目・各7段階）

| 列名 | 1（左） | 7（右） | ねらい |
|------|--------|--------|--------|
| sd_like | 好ましくない | 好ましい | 総合的な好み |
| sd_character | 味気ない | 趣のある | 「味」の核 |
| sd_clean | 汚い | きれいな | 「汚れ・劣化」の核 |
| sd_warm | 冷たい | 温かい | 木の温かみ（感性） |
| change | 変化なし（基準と同じ） | 大きく変化（経年） | 基準からの変化量 |

「基準からの変化（change）」は**良し悪しではなく変化の大きさだけ**を答える項目。良い変化か悪い変化かは他項目（好ましい・趣のある・きれいな）で測り，解析時に切り分ける。

### 3.2 途中でやめる・再開する

- 進行状況は**ブラウザ内（自動保存）**に常時保存される。加えて「途中保存して中断」や完了時に**CSVが出力**され，このCSVからも再開できる。
- **再開は2通り：**
  1. **同じPC・同じブラウザ**：このHTMLを開き直す → 上部「前回の続きがあります」→ **「続きから再開」**。PCを再起動しても自動保存は残る。誤ってタブを閉じた場合も同じ。
  2. **別PC・自動保存が消えた場合**：中断時または完了時に保存したCSVを，画面上部の **「保存したCSVから再開」** で読み込む → 未回答の続きから再開できる。
- 中断時のCSVには**未回答ぶんも空欄の行**として含まれる（再開に使うため）。解析時は回答済み行だけ使えばよい。同じ評価者の完了時CSVが完全版になる。
- 同じ人が再開する場合は，途中で別の評価者のセッションを始めない（ブラウザ内の自動保存が上書きされるため。なお，CSVから再開すればこの影響は受けない）。

## 4. データの回収とまとめ

### 4.1 出力されるCSV

- ファイル名：`SDtag_<評価者ID>_<日時>.csv`（中断時は `SDtag_<評価者ID>_partial_<日時>.csv`）。
- 文字コードはBOM付きUTF-8（Excelでそのまま開ける）。

### 4.2 回収

- **共有1台方式**：そのPCのダウンロードフォルダに人数ぶんのCSVが溜まる。
- **各自PC方式**：各評価者から自分のCSVを送ってもらう。
- まとめは，集まった個別CSVを**そのまま連結**すればよい（同じ列構成）。`rater_id` で人を区別する。

### 4.3 進捗確認（あと何人分必要か）

ツールの「進捗確認」パネルで，集めたCSVを**複数まとめて選択**して読み込むと，各画像が何人に評価されたか，目標Rに対して**あと約何人分必要か**が表示される（練習は自動除外）。実施の途中段階で「もう何人集めればよいか」を判断できる。

### 4.4 CSVの列（解析用）

| 列名 | 意味 |
|------|------|
| rater_id | 評価者ID |
| session_start | セッション開始時刻 |
| order | その評価者の提示順（0始まり） |
| specimen | 試料（SC-2 / SC-4） |
| week | 経過週（0〜104） |
| set_id | セット指定モードのセット番号（他モードは空） |
| is_repeat | 1=一貫性チェックの再掲 |
| is_practice | 1=練習（**解析から除外する**） |
| rt_ms | その画像の回答所要時間（ミリ秒） |
| sd_like / sd_character / sd_clean / sd_warm / change | 各評価値（1〜7） |

---

## 5. 解析の指針（参考）

- **練習を除外**：`is_practice=1` の行は集計・解析から外す。
- **一貫性（再現性）**：同一 `rater_id`×`specimen`×`week` で `is_repeat` の1回目と2回目を突き合わせ，評価値の差で評価者内のブレを確認する。差が大きい評価者は要注意。
- **味と汚れの分離**：`change`（変化量）と `sd_like`・`sd_character`・`sd_clean` をクロスし，「変化大×好ましい・趣がある＝味のある経年」「変化大×汚い・好ましくない＝劣化・汚れ」を切り分ける。主成分分析・因子分析も有効。
- **暴露量との対応**：`week` をキーに，別途用意した気象の累積暴露量（累積日射量・累積降水量など）と結合すると，印象の変化と日照・降雨の関係を調べられる。

## 6. 注意・トラブルシュート

| 症状 | 対処 |
|------|------|
| 画像が表示されない（×印） | `タグ付けツール.html` と `SC-2_cropped/`・`SC-4_cropped/` が同じ場所にあるか確認。HTML単体だけ配布していないか確認 |
| 開いたら他人の「続きから再開」が出る | 別人が始めるときは「破棄して新規に始める」を押してから開始 |
| 別PCに移ったら再開できない | 仕様（保存はブラウザ内）。中断時の partial CSV を使う |
| Excelで文字化け | BOM付きで出力済み。直らない場合は「データ取得→テキスト/CSV→UTF-8」で開く |
| 全員ぶんを1ファイルに自動集約したい | 純粋なローカルHTMLでは不可。簡易サーバや共有フォームが必要（要相談） |

## 付録：評価者への教示（読み上げ用の例）

> これから木材の写真を1枚ずつ見ていただきます。左に「参照基準」（設置した直後・未暴露の状態）、中央に「評価対象」が出ます。評価対象が、基準からどれくらい変化・経年して見えるか、また好ましさや汚れ・温かみなどの印象を、各項目7段階で答えてください。深く考え込まず、見たままの第一印象でかまいません。最初の2枚は練習です。同じ写真が複数回出ることがありますが、その都度その時の印象で答えてください。途中で約10枚ごとに休憩が入ります。途中でやめたいときは「途中保存して中断」を押してください。
